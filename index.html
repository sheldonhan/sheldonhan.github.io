如何做好架构设计？

架构设计三原则介绍

  • 原则1：合适原则

    • 宣言：合适优于业界领先
    • 资源 + 时间 + 业务

  • 原则2：简单原则

    • 宣言：简单优于复杂
    • 可靠性 + 可扩展 + 故障处理效率

  • 原则3：演化原则

    • 宣言：演化优于一步到位
    • 创造 + 迭代优化 + 重构重写

架构设计三原则案例

架构设计三原则应用

  • 设计出来的架构要满足当时的业务需要，符合团队和技术的能力水平（合适原则）
  • 先按照简单的方式来设计架构，然后不断地在实际应用过程中迭代优化（简单原则）
  • 当业务发生变化时，架构要扩展、重构，甚至重写（演化原则）
  • 合适 > 简单 > 演化

架构设计原则常见的判断维度

  • 业务 + 团队 + 技术

什么是面向复杂度架构设计？

架构设计方法论

  • 面向模式软件架构
  • 风险驱动架构设计
  • DDD领域驱动设计：DDD和整洁架构

面向复杂度的架构设计

为什么做架构设计？

  • 核心原因：软件系统规模增长
  • 核心特点：数据结构和算法不再是主要问题，整个系统的结构成为首要问题。

怎么做架构设计？

  • 思路：通过分析系统需求找到系统复杂的地方，然后设计方案

有哪些复杂度？

  • 模式：复杂度来源：高性能、高可用、可扩展、安全、成本

怎么降低复杂度？

  • 套路：分库分表、缓存、集群、分片、微服务、DDD、异地多活……

架构设计环

  • 需求 -> 判断 -> 复杂度 -> 拆解 -> 备选架构 -> 取舍 -> 架构方案(4R) -> 实现
